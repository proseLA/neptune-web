OverlayHeader Upgrade Guide

January 22, 2021
    5 months ago by Helen Durrant

    See the closely related FlowNavigation upgrade guide.

    OverlayHeader has gone through a major refactor, and there are some breaking changes. This upgrade guide will provide the necessary steps to switch to the new version.

    API Changes

    Old APINew APIChange
    avatar: Types.shape({url: Types.string})avatar: Types.elementInstead of an object with a URL, we now expect an Avatar.
    onCloseonCloseNo change
    -logo@new We used to display the logo within the component using CSS. We now expect you to pass in a logo as an image. You will need to use a light logo if working on a dark screen.
    theme-@deprecated Used to change logo appearances and borders. Now you can pass and manage logo.
    profileType-@deprecated It used to manage the type of Avatar. Since now avatar is provided from outside you can decide which type of Avatar youâ€™d like to pass (personal or business) in your app.

    Dependency requirements

    @transferwise/components >= 34.2.3

    @transferwise/neptune-css>= 8.0.1

    CSS changes

    Dimensions

    OverlayHeader will now occupy 100% of its parent so please provide a container to make it fit to your design.

    Border

    Previously the bottom border was handled by the component, now consumers are required to provide it.
