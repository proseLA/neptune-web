Neptune CSS provides styles for Neptune React & Angular components, loads of CSS utility classes and LESS Variables/CSS Custom Properties (for colors, spacing, font size and more).

## Getting started

Refer to the <DocumentLink href="/about/Setup">setup guide</DocumentLink> for information on how to install and import Neptune CSS into your application.

## Usage

Refer to the <DocumentLink href="/styles/core/Typography">documentation</DocumentLink> for information on available utility classes.

## Tokens (CSS Custom Properties)

We maintain a [separate package](https://www.npmjs.com/package/@transferwise/neptune-tokens) where we define platform agnostic variables like spacing and colour codes.

These are imported in `neptune.css` and are available at root level as <Link href="https://developer.mozilla.org/en-US/docs/Web/CSS/--*" target="_blank">CSS Custom Properties (aka CSS Variables)</Link> which you can use in your codebase.

Information on what tokens are available can be found in the <DocumentLink href="/styles/tokens/Colors">tokens documentation</DocumentLink>.

Usage example:

```css
.my-button {
  background-color: var(--color-background-accent);
  border: 1px solid transparent;

  color: var(--color-content-primary);
  cursor: pointer;

  padding: var(--size-8);
}
```

**Note:** if you need support old browsers (see <Link href="https://developer.mozilla.org/en-US/docs/Web/CSS/--*#browser_compatibility" target="_blank">browser compatibility table</Link>) you need to do some setup to ensure fallbacks are correctly generated.

We suggest you use <Link href="https://postcss.org/" target="_blank">PostCSS</Link>, in particular <code><Link target="_blank" href="https://www.npmjs.com/package/@arshaw/postcss-custom-properties">@arshaw/postcss-custom-properties</Link></code> plugin which enables fallbacks not only for old browsers, but for modern browsers too (in the case of a missing or misspell custom property).

```js
// plugin setup in postcss.config.js
require('@arshaw/postcss-custom-properties');

module.exports = {
  plugins: {
    '@arshaw/postcss-custom-properties': {
      importFrom: ['../../node_modules/@transferwise/neptune-css/dist/props/neptune-tokens.css'],
      preserveWithFallback: true,
    },
    ...
  },
};
```

Optional highly but recommended is to set up [Stylelint](https://stylelint.io) in your repository, and configure the [stylelint-value-no-unknown-custom-properties](https://github.com/csstools/stylelint-value-no-unknown-custom-properties) plugin to similarly point to the custom properties files, to ensure you are not using any that don't exist.

## Migrating from Bootstrap

Previously, apps at Wise were styled using a fork of <Link target="_blank" href="https://bootstrap.transferwise.com/">Bootstrap</Link>. Neptune CSS is based heavily off this fork, with major modifications. We want to move everyone off Bootstrap and on to Neptune CSS, where we will continue to make improvements.
